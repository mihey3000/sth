# Галерея: как обновлять контент

Этот проект публикуется как статический сайт через `npm run generate`.
Галерея собирается автоматически на этапе сборки и не требует бэкенда.

## Как добавить новые фотографии

1. Поместите изображения в любую папку в корне проекта (рядом со `sth/`).
2. Не используйте папки `Photos`, `images` и `sth` — они исключены из галереи.
3. Поддерживаются форматы: `jpg`, `jpeg`, `png`, `webp`.
4. Запустите сборку: `npm run generate`.

## Локальная разработка

Если нужно увидеть актуальную галерею при `npm run dev`, выполните:
1. `npm run pregenerate` — обновит `gallery-manifest.json` и WebP в `public/gallery`.
2. Затем запустите `npm run dev`.

После сборки изображения будут оптимизированы и сохранены в `sth/public/gallery`
в формате WebP, а список будет сохранен в `sth/public/gallery-manifest.json`.

## Как работает генерация

Генератор находится в `sth/scripts/build-gallery.mjs` и выполняется автоматически
перед `npm run generate` (через `pregenerate`).

Он делает следующее:
- сканирует корень проекта;
- исключает папки `Photos`, `images`, `sth`, а также `node_modules` и `.git`;
- собирает только изображения;
- оптимизирует изображения и сохраняет их в `sth/public/gallery` (оригиналы не меняются);
- создает `sth/public/gallery-manifest.json`, который читает страница галереи.

## Очистка галереи вручную

Чтобы полностью очистить оптимизированные изображения и манифесты, используйте:

`npm run clean:gallery`

Эта команда удаляет `sth/public/gallery` и сбрасывает манифесты в `[]`.

## Где используются данные

Страница `sth/pages/gallery.vue` читает `gallery-manifest.json` и строит:
- список карточек;
- категории (по имени папки);
- пагинацию "Показать еще".

Если нужно изменить правила формирования названия/описания/категорий,
правьте `sth/scripts/build-gallery.mjs`.
